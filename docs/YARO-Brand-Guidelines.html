5 Core Logic & Animation Principles

Architectural Blueprint
This section stores concepts and parameters behind YARO's core interactions — not raw code. Use these principles as a lightweight guide for building or recreating features.

1. Cinematic Slide Orchestration ("Zero-Cut" Principle)

Where: goToSlide() — index.html portal, gallery, and offer transitions

No abrupt visual cuts. Every transition uses blur crossfade with subtle scale shifts.

  • Ease: power2.inOut or power3.inOut for slide-level transitions
  • Blur: Fading out = scale(0.85) + blur(12px); coming in = reverse
  • Loop (slide 2→0): 1.1s sine.inOut, inner content blur(25px) + scale(0.45) + y drift
  • Layer isolation: Keep target elements separate from parent transforms to avoid browser clipping artifacts
  • Safety: isTransitioning reset if stuck >3s

2. Handcrafted Logo Scatter (GSAP Hover)

Where: #yaro-title — index.html hero title mouseenter/mouseleave

Asymmetrical offsets create a premium physical "explosion" rather than random scattering.

  • Y: (x: -24, y: -6, rot: 18) — high, curves left
  • A: (x: -8, y: -18, rot: 8) — very high, slight left
  • R: (x: 8, y: -18, rot: -8) — very high, slight right
  • O: (x: 24, y: -6, rot: -18) — high, curves right
  • Container: rotationX: -10, rotationY: 5, ease power3.out, 0.9s
  • Badge push: All .premium-badge elements shift down 24px during scatter

3. Impulse Border — @property Animation

Where: .cta-btn::before on index.html

Uses CSS @property to register a custom angle variable, animated via @keyframes — avoids transform:rotate() fan-blade artifacts.

  • --impulse-angle: Registered as <angle>, animated 0→360deg (4s linear, 1.5s on hover)
  • Impulses: White at 0°, violet at 180°, with soft falloff
  • Ring: 3px visible via inset: -1px; padding: 3px; mask-composite: exclude
  • Glass sweep: ::after diagonal sheen, 4s (75% idle, then sweep)
  • Sparks: 3 particles per side, burst at 25%/75% cycle, staggered 1-2%
  • Magnetic pull: JS mouse-follow clamped to 8px max, rAF-driven

4. Cursor Interaction & Trail System

Where: index.html, store.html, cart.html

Three-layer cursor: precise dot + lagging ring + canvas trail for particle-like afterglow.

  • Dot: 3px white, lerp 0.25, turns violet on interactive elements
  • Ring: 24px ghost ring with conic violet gradient (lavaSpin 2.5s), lerp varies by page
  • Trail: 18-point history on canvas, violet decay (rgba(167,139,250, life*0.25)), lineWidth life*1.2
  • States: .hovering (violet dot, 44px ring), .on-logo, .clicking, .hidden
  • Interactive: Class .interactive-elem toggles hover states via mouseenter/mouseleave
  • Mobile: Hidden at ≤820px or hover: none
  • Negative inversion: collection filter button uses mix-blend-mode: difference, 50px white ring

5. Desktop Gallery — Circuit Wire & Pulse Sequence

Where: #slide-2 on index.html, screens >820px

High-voltage aesthetic: SVG circuit wires with traveling sparks, then pulse rings, then card pop.

  • Sparks: 4 SVG paths with stroke-dashoffset 30→-200 (0.38s, staggered 0.03s)
  • Pulse rings: Two layered rings scale from 0.95 to 1.2-1.35 with blur, then fade
  • Card pop: Side cards get .pop-straight class for 1.5s (brightened, un-blurred, elevated z-index)
  • Timing: All triggered from goToSlide(1) timeline at offsets 1.4s→2.0s→3.5s

6. Mobile Gallery — 1-by-1 Showcase

Where: #slide-2 .product-gallery at ≤820px

Replaces the desktop 3-card layout with a single-card carousel that auto-cycles every 2s.

  • Center card: scale(0.88), full opacity, no blur
  • Side cards: scale(0.55), blur(6px), opacity(0.45)
  • Transition: power2.out, 0.5s with staggered blur
  • Loop: setInterval cycling currentIdx = (currentIdx + 1) % 3

7. Slide 3 — Featured Stories Auto-Cycle

Where: #slide-3 — desktop + mobile

3 story items highlight in sequence, then advance to the next slide (no looping within the slide).

  • Highlight: item 0 (immediate) → 1.8s → item 1 → 1.8s → item 2 → 0.6s → goToSlide(currentSlide + 1)
  • Image: Updates featured image + desktop title + mobile title on each step
  • Transition: GSAP fromTo opacity 0.3→1, blur 4px→0, scale 0.98→1 (0.2s power2.out)
  • Cleanup: Timer cleared when leaving slide 3

8. Header Search — Cross-Page Sync

Where: .sh-search in header on store.html, product.html, cart.html

Single search bar connects to the product catalog across all pages.

  • Navigation: On Enter → store.html?search=<query>
  • Local override: If on store.html, fills page's search input and dispatches input event — no reload
  • URL param init: Reads ?search= on page load, triggers filter
  • Clear button: × appears when text exists, clears on click

9. Product Filtering — Category + Search AND Logic

Where: store.html — updateProducts()

Dual-axis filtering with AND logic: category selection AND text search work simultaneously.

  • Categories: 12 categories — each product card has data-categories attribute
  • Search: Matches against product name AND all category tokens
  • AND logic: categoryMatch && searchMatch must both pass
  • Heading: "Search: query" / category name / "Best Selling"
  • Count: Live display of visible items (e.g. "5 items")
  • SPA-style: history.pushState updates URL without page reload

10. Multi-Bucket Cart & Checkout

Where: cart.html, product.html

LocalStorage-based cart with up to 3 named baskets, promo codes, and full checkout flow.

  • Storage: localStorage.yaro_buckets — array of { name, items: [] }
  • Bucket tabs: Click to switch, inline rename, "Add Basket" (max 3)
  • Item ops: Quantity +/- (stock limits), remove with confirm modal
  • Promo codes: YARO10 (-10%), WELCOME (-15%), FLASH20 (-20%) — case-insensitive
  • Free shipping: ₹699 threshold with gradient progress bar
  • Checkout: Address validation (name, phone, pincode, city, state) → order ID gen → cart clear → success modal (3-5 day ETA)
  • Urgency timer: 15:00 countdown, resets at 0:00

11. Product Detail — Gallery & Actions

Where: product.html

Dynamic product detail with image gallery, size selector, and two distinct action animations.

  • Image gallery: Main showcase + 3 thumbnails + arrow nav. GSAP: opacity 0.3→1, scale 0.92→1, blur 4px→0 (0.45s). Mobile swipe (40px threshold).
  • Size buttons: S-XXL, liquid swelling effect via ::before scale animation (0.6s elastic.out)
  • "Keep This": Inner sparkle chase — ::before background-position sweep, 2.8s infinite
  • "Add to Cart": Purple circle swells from center (::before scale 0→20), 0.75s on hover
  • Related carousel: Drag/swipe + arrow nav with GSAP spring bounds

12. Customer Reviews System

Where: product.html

Seeded reviews, interactive star rating, and expandable review modal.

  • Seeded data: 8 hardcoded reviews with names, dates, ratings, text
  • Star rating: Hover brightens stars, click sets rating (1-5)
  • Form: Stars + textarea with 500 char limit and live counter
  • Submit: Validates rating + text → adds to list → toast notification
  • "See More": Toggle to expand/collapse long reviews
  • Review modal: Click any review card → full content overlay

13. Toast Notification System

Where: cart.html, product.html

Non-intrusive slide-in notifications for cart actions, promo codes, and errors.

  • Position: Fixed top-right, stacked vertically
  • Entrance: translateX(120%) → 0, 0.4s ease
  • Auto-dismiss: 3.5s, or click to dismiss immediately
  • Types: .success (green #22c55e), .error (red #ef4444)
  • Usage: "Added to cart", "Removed", "Promo applied", "Order placed", error messages

14. Leaflet Map Integration

Where: cart.html — delivery location section

Dark-themed Leaflet map with geolocation and draggable pin.

  • Tiles: CartoDB dark matter via unpkg CSS
  • Marker: Draggable, shows lat/lng on move
  • Click-to-drop: Click anywhere on map to move marker
  • Geolocation: navigator.geolocation.getCurrentPosition on load, high accuracy
  • Fallback: Defaults to New Delhi [28.6139, 77.2090]

15. Page Entrance & Transition Patterns

Where: All pages

Consistent entrance and exit patterns across all page navigations.

  • Index entrance: Glow sweep (0.6s) → logo drops from blur 20px, y -80, scale 0.7 (1.3s power4.out) → subtitle fade → CTA fade
  • Collection entrance: GSAP stagger on .reveal-item (opacity, y, scale, blur)
  • Product entrance: TranslateY slide overlay uncovers. Gallery slides left (0.65s), info slides right (0.65s)
  • Cart entrance: GSAP stagger fade-up on frame-content children
  • Exit (all): Content scale(0.96) + opacity + blur → overlay active → navigate after 250ms

Easing Curve Reference

Shared across all pages
  • cubic-bezier(0.22, 1, 0.36, 1) — Primary GSAP-style ease (cursor, CTA, modals, toasts)
  • cubic-bezier(0.19, 1, 0.22, 1) — Buttons, cards, thumbnails, search expand
  • cubic-bezier(0.16, 1, 0.3, 1) — Modal entrances (category modal)
  • cubic-bezier(0.4, 0, 0.2, 1) — Page transitions (index/collection opacity overlay)
  • cubic-bezier(0.76, 0, 0.24, 1) — Page transition (product.html translateY slide)
  • power2.out / power3.out / power4.out — GSAP animations
  • elastic.out(1, 0.3) — Size button bounce feedback

Glass Morphism Pattern (All Pages)

Architecture
  • Background: Multiple radial gradients (violet/purple atmospheric tones) over #0A0A0A
  • Border bevel: rgba(255,255,255,0.22) top, rgba(0,0,0,0.3) bottom — creates 3D rim
  • Box shadows: 3 outer drop-shadows, 2 outer bevel, 4 inner specular/depth, 1 violet ambient rim
  • Grid overlay: ::after — 1px lines at 80px intervals (60px on product.html)
  • Noise texture: ::before — SVG feTurbulence, mix-blend-mode: overlay, 0.35-0.4 opacity
